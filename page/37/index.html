    ExtJS Class System

    命名规则Class System Class names may only contain alphanumeric characters. Class names should be grouped into packages where appropriate and properly namespaced using object property dot-notation. MyCompany.data.CoolProxy MyCompany.Application The top-level namespaces and the actual class names should be in CamelCased, everything else should be all lower-cased MyCompany.form.action.AutoLoad Classes that are not distributed by Sencha should never use Ext as the top-level namespace. Acronyms should also follow CamelCased convention listed above Ext.data.JsonProxy instead of Ext.data.JSONProxy As a result, there must only be one class per file. Static class properties that are constants should be all upper-cased. Ext.MessageBox.YES = "Yes" Ext.MessageBox.NO = "No" MyCompany.alien.Math.PI = "4.13" Continue reading...
